TypeScript

1. What is TypeScript?

TypeScript is a superset of JavaScript developed by Microsoft. It adds static typing, interfaces, and compile-time error checking. TypeScript code is written in .ts files and compiled into JavaScript. It helps identify errors before runtime, improving code reliability.

2. Key Features of TypeScript

a) Static Typing

let age: number = 20;
let name: string = "Vaibhav";

b) Type Inference

let x = 10; // inferred as number

c) Interfaces

interface Student {
  name: string;
  age: number;
}

d) Classes and Object-Oriented Programming

Supports classes, inheritance, and access modifiers (public, private, protected).

e) Generics

function identity<T>(value: T): T {
  return value;
}

f) Enums

enum Direction { Up, Down, Left, Right }

3. Advantages of TypeScript

  • Detects errors during development (before execution).
  • Improves code quality and maintainability.
  • Provides better tooling support (auto-completion, refactoring).
  • Enables object-oriented programming features.
  • Fully compatible with existing JavaScript libraries.

4. Compilation Process

tsc file.ts
// Generates file.js

5. Basic Data Types in TypeScript

  • number
  • string
  • boolean
  • any
  • unknown
  • void
  • null, undefined
  • array (e.g., number[])
  • tuple
  • enum

6. TypeScript vs JavaScript

JavaScript TypeScript
Dynamically typed Statically typed
Errors detected at runtime Errors detected at compile time
No type checking Strong type checking
Only supports JavaScript features Extends JavaScript with extra features

7. Uses of TypeScript

  • Used in frameworks such as Angular (default language).
  • Commonly used with React and Node.js.
  • Suitable for large projects where maintainability and type safety are important.
